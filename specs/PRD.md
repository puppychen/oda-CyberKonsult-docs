## Epic 1：使用者認證與授權（FR-01）🔄

### US-01-01：帳號密碼登入

> 身為**使用者**，我希望能夠使用帳號密碼登入系統，以便存取我有權使用的功能。

| AC 編號 | 驗收條件 |
|---------|----------|
| AC-01-01-01 | 支援 Email + 密碼登入，登入成功後回傳 JWT access token + refresh token |
| AC-01-01-02 | 登入失敗顯示「帳號或密碼錯誤」，不洩漏具體失敗原因 |
| AC-01-01-03 | 密碼錯誤超過 5 次鎖定帳號 15 分鐘，回傳 423 狀態碼 |
| AC-01-01-04 | 密碼符合資通安全「普」級政策：8 碼以上、含大小寫+數字+特殊字元、90 天過期、歷史 2 代不重複 |

### US-01-02：使用者帳號管理

> 身為**系統管理員**，我希望能夠管理使用者帳號，以便控制系統存取權限。

| AC 編號 | 驗收條件 |
|---------|----------|
| AC-01-02-01 | 可透過 Admin Dashboard 建立新使用者 |
| AC-01-02-02 | 可停用/啟用帳號，停用後使用者無法登入 |
| AC-01-02-03 | 可指派角色（user / consultant / data_cleaner / admin） |

### US-01-03：安全登出

> 身為**使用者**，我希望能夠安全登出系統，以確保帳號安全。

| AC 編號 | 驗收條件 |
|---------|----------|
| AC-01-03-01 | 登出後 Token 失效，無法再使用 |
